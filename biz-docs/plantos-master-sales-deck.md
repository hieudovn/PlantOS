## Slide 2 — The Industrial Data Challenge

# Plants generate data. Enterprises still lack operational intelligence.

Industrial data is often:

- fragmented across PLC, SCADA, DCS, historian, MES, EAM and ERP
- organized around raw tags, not assets and operations
- difficult to reuse across applications
- hard to govern across lines, plants and sites
- not ready for advanced analytics or AI

**The problem is not data availability. It is data usability.**

---

## Slide 3 — Why Existing Approaches Are Not Enough

# More dashboards do not solve the data foundation problem.

| Existing approach | Limitation |
|---|---|
| SCADA / DCS | Built for control, not enterprise data reuse |
| Historian | Stores time-series, but usually lacks full business context |
| Dashboard tools | Visualize values, but often hard-code tags |
| Data lake | Stores data, but does not create operational meaning |
| Point-to-point integration | Fast at first, complex at scale |
| Standalone AI tools | Need trusted context before they can reason |

**Without a shared operational model, every new use case starts again from raw data.**

## Slide 9 — UNS + CDM: Two Complementary Roles

# Address the data. Define the meaning.

| Layer | Role |
|---|---|
| Unified Namespace | Standard operational address space |
| Canonical Data Model | Shared application data contract |
| Asset Model | Stable identity and hierarchy |
| Signal Model | Semantic measurement definition |
| Event Model | Operational timeline and state changes |
| Context Layer | Production, maintenance, quality and energy meaning |

**UNS organizes how data flows. CDM defines what the data means.**
